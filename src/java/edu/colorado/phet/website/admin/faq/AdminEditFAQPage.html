Instructions

A FAQ page is essentially composed of a list of headers and questions. Here you can add headers and questions, edit them, reorder them, or delete them. For simplicity, I call both headers and questions "items".

Each item will have a name. This name is used for multiple purposes:

  • You can make a link to specific headers or questions by using their name after a "#". For example, http://phet.colorado.edu/(....)/some-faq#molecules points to the item with the name "molecules". When clicked on, this link will go to that FAQ, and scroll the screen so that the item selected is visible.
  • The name is also used in the translation keys that are needed for the underlying system and for translators. Thus, if you have a FAQ for Build a Molecule and a question named "how-to-do-X", it will create translatable strings with the keys "faq.build-a-molecule.how-to-do-X.question" and "faq.build-a-molecule.how-to-do-X.answer".

Names should only contain letters (lower case preferred, but upper case is allowed), digits, and hyphens. Additionally, after creating an item, it's name cannot be changed. If you do not enter in a name, it will attempt to create one based on either the question or header specified. No two items in the same list should have the same name!

Any text can be edited at any time after creating a question or header. Just click on the text you want to change, and it should turn into a text-editing box. Make your changes, and then click outside to immediately save the change.

A preview is available to the right side to show the immediate effect of changes.

It is recommended that answers use XHTML formatting to display their responses. This means using <strong> and <em> instead of <b> and <i>, and following all best practices. If you have any questions, please do not hesitate to ask the website administrator. (Also note that as of right now, newlines in the edit box are turned to line-breaks in HTML (<br/>), so don't split lines inside of <ul> tags).

It is recommended to validate after changes, since it is very hard to scan through an entire FAQ to find a problem. In the future, it may be possible to identify the question/header that has the problem.
